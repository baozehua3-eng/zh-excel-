# 📚 完整部署操作指南

## 🎯 目标
1. 将代码上传到 GitHub（代码托管）
2. 部署到 Vercel（让所有人可以在线访问）

---

## 📋 第一步：准备 Git 仓库

### 1.1 打开终端

在项目文件夹 `c:\Users\HXC369\Desktop\cursor\test1` 中：
- **方法A**：按住 `Shift` 键，右键点击文件夹空白处，选择"在此处打开 PowerShell 窗口"
- **方法B**：在 VS Code 中按 `Ctrl + ~` 打开终端

### 1.2 初始化 Git 仓库

在终端中依次执行以下命令：

```bash
# 步骤1：初始化 Git
git init

# 步骤2：添加所有文件（.gitignore 会自动排除 node_modules 等）
git add .

# 步骤3：提交代码
git commit -m "Initial commit: Excel文件对比工具"
```

**预期结果**：看到类似 "X files changed" 的提示，说明提交成功。

---

## 📋 第二步：创建 GitHub 仓库

### 2.1 登录 GitHub

1. 打开浏览器，访问：https://github.com
2. 如果没有账号，点击 "Sign up" 注册（免费）
3. 如果已有账号，点击 "Sign in" 登录

### 2.2 创建新仓库

1. 登录后，点击右上角的 **"+"** 图标
2. 选择 **"New repository"**（新建仓库）

3. 填写仓库信息：
   - **Repository name**（仓库名称）：输入 `excel-compare-tool`（或你喜欢的名字）
   - **Description**（描述）：可选，例如 "Excel文件对比工具"
   - **Public** 或 **Private**：选择 Public（公开）让其他人可以看到
   - ⚠️ **重要**：**不要**勾选以下选项：
     - ❌ "Add a README file"
     - ❌ "Add .gitignore"
     - ❌ "Choose a license"
   - 保持所有选项为空

4. 点击绿色的 **"Create repository"** 按钮

### 2.3 获取仓库地址

创建成功后，GitHub 会显示一个页面，上面有仓库地址，类似：
```
https://github.com/你的用户名/excel-compare-tool.git
```
**复制这个地址，下一步会用到！**

---

## 📋 第三步：推送代码到 GitHub

回到你的终端窗口，执行以下命令：

```bash
# 步骤1：添加远程仓库（替换为你的实际仓库地址）
git remote add origin https://github.com/你的用户名/excel-compare-tool.git

# 步骤2：设置主分支名称
git branch -M main

# 步骤3：推送代码到 GitHub
git push -u origin main
```

**注意事项**：
- 如果是第一次使用 Git，可能会要求输入 GitHub 用户名和密码
- 如果提示输入密码，需要使用 **Personal Access Token**（不是 GitHub 密码）
  - 获取 Token：GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token
  - 权限勾选：`repo`
  - 复制生成的 token，用它作为密码

**预期结果**：看到类似 "Enumerating objects... Writing objects... done" 的提示，说明推送成功！

---

## 📋 第四步：部署到 Vercel（让所有人可以访问）

### 4.1 访问 Vercel

1. 打开浏览器，访问：https://vercel.com
2. 点击 **"Sign Up"** 或 **"Log In"**
3. 选择 **"Continue with GitHub"**（使用 GitHub 账号登录，最简单）

### 4.2 导入项目

1. 登录后，点击 **"Add New..."** 按钮
2. 选择 **"Project"**
3. 在项目列表中，找到你刚才创建的 `excel-compare-tool` 仓库
4. 点击 **"Import"** 按钮

### 4.3 配置部署

Vercel 会自动检测到这是 Vite 项目，通常不需要修改配置：

- **Framework Preset**：应该是 "Vite"（自动识别）
- **Root Directory**：保持默认（`./`）
- **Build Command**：`npm run build`（自动填充）
- **Output Directory**：`dist`（自动填充）

**直接点击 "Deploy" 按钮！**

### 4.4 等待部署完成

1. 部署过程大约需要 **1-2 分钟**
2. 你会看到构建日志在实时更新
3. 看到绿色的 **"Success"** 提示时，部署完成！

### 4.5 获取访问链接

部署完成后，你会看到一个链接，例如：
```
https://excel-compare-tool.vercel.app
```

**这个链接就是你的在线工具地址！** 可以分享给任何人使用。

---

## 📋 第五步：后续更新代码（迭代开发）

以后当你修改了代码，想要更新在线版本时：

### 5.1 提交修改

在终端中执行：

```bash
# 1. 查看修改了哪些文件
git status

# 2. 添加所有修改的文件
git add .

# 3. 提交修改（描述你做了什么）
git commit -m "修复了某个bug"  # 或 "添加了新功能"

# 4. 推送到 GitHub
git push
```

### 5.2 自动部署

**Vercel 会自动检测到 GitHub 的更新，并自动重新部署！**

- 通常 1-2 分钟后，你的在线版本就会自动更新
- 可以在 Vercel 控制台看到部署状态

## 🆘 常见问题

### Q1: Git push 时提示需要认证
**A**: 使用 Personal Access Token 作为密码，不是 GitHub 登录密码。

### Q2: Vercel 部署失败
**A**: 检查：
- `package.json` 中的 `build` 脚本是否正确
- 是否有语法错误（运行 `npm run build` 测试）

### Q3: 如何修改部署后的链接
**A**: 在 Vercel 项目设置中可以自定义域名。

### Q4: 如何回退到之前的版本
**A**: 在 Vercel 的部署历史中，可以点击任意版本重新部署。
